Zo lees je dit document

Per grafiek lopen we de code van boven naar beneden door in kleine blokjes. Onder elk blokje staat per regel wat hij doet — en waar nuttig een Aanpassen-tip.

  • Code-blokken zijn de échte regels uit het bestand (donkere achtergrond).
  • De nummers links in een blok verwijzen naar het regelnummer in het bronbestand.
  • Aanpassen = een concrete knop waar je aan kunt draaien (kleur, snelheid, aantallen…).
Terugkerende begrippen — const/let = een waarde een naam geven (const = vast, let = mag wijzigen). => = een "pijl-functie" (kort recept). d3 = de tekenbibliotheek. state = de gedeelde data. $('#id') = zoek het HTML-element met dat id. Een volledige begrippenlijst staat achterin.

1 · aquarium.js canvas

src/scripts/mitchell/charts/aquarium.js

Tientallen visjes zwemmen rond op één <canvas>. Aantal per soort = hoe vaak gezien; grootte = gewicht. Klik = schrikken + rimpel, hover = naam & lengte.

1.5 — Vis-plaatjes inkleuren: de sprite-cache (regel 48–70)

49   const sprites = {}, imgCache = {};
51   function loadImage(url) {
52     if (imgCache[url]) return imgCache[url];   // al geladen? hergebruik
53     const img = new Image(); img.src = url; imgCache[url] = img; return img;
54   }
58   function makeSprite(naam, color) {
59     const key = naam + color;
60     if (sprites[key]) return sprites[key];   // al ingekleurd? hergebruik
61     const img = loadImage(fishImagePath(naam));
62     if (!img.complete || !img.naturalWidth) return null; // nog niet binnen
66-70   // teken de PNG op een mini-canvas 'c' van 64px breed
  • 49 Twee "kaartenbakken": sprites (ingekleurde resultaten) en imgCache (rauwe afbeeldingen). Cachen = werk maar één keer doen.
  • 51–54 loadImage: is de afbeelding al geladen, geef 'm terug; anders start het laden en onthoud 'm.
  • 59–60 De sleutel is naam+color; bestaat die al, klaar. Zo krijgt elke soort+kleur precies één ingekleurde versie.
  • 62 Is de PNG nog niet binnen (complete/naturalWidth), geef null terug — de tekenlus probeert het dan volgende frame opnieuw.
  • 66–70 Teken de foto op een onzichtbaar mini-canvas c (basisbreedte 64px, hoogte volgens beeldverhouding).

1.6 — Pixel-voor-pixel inkleuren (regel 76–93)

76     const L = 0.4, lift = 1.12;
77     const [r0, g0, b0] = hexToRgb01(color);
78-80   const tr = r0 + (1 - r0) * L; // kleur richting wit mengen (lichter)
82     const data = offCtx.getImageData(0, 0, pixW, pixH);
83     const arr = data.data;       // [r,g,b,a, r,g,b,a, ...]
84     for (let i = 0; i < arr.length; i += 4) {
85       if (arr[i + 3] === 0) continue;  // transparant → overslaan
86       const y = 0.299*arr[i] + 0.587*arr[i+1] + 0.114*arr[i+2]; // grijswaarde
87-89   arr[i] = Math.min(255, y*tr*lift); // grijs × doelkleur
91     offCtx.putImageData(data, 0, 0); sprites[key] = c; return c;
  • 76 L = hoeveel de kleur richting wit gemengd wordt (lichter); lift = extra helderheid. Hogere waarden = lichtere visjes.
  • 77 Zet de hex-kleur om naar drie getallen 0–1 (rood, groen, blauw).
  • 78–80 Meng elke kleurcomponent richting wit: kleur + (1−kleur)·L.
  • 82–83 Lees alle pixels uit. arr is één lange rij getallen: per pixel 4 waarden (rood, groen, blauw, doorzichtigheid).
  • 84 Loop met stappen van 4 (één pixel per keer).
  • 85 Volledig doorzichtige pixel? Overslaan (achtergrond blijft leeg).
  • 86 Bereken de grijswaarde y met de standaard ogen-weging (groen telt het zwaarst). Zo behouden we licht/donker = textuur.
  • 87–89 Vermenigvuldig die grijswaarde met de doelkleur → de vis krijgt zijn kleur maar houdt zijn schaduwen. Math.min(255,…) voorkomt te-felle pixels.
  • 91 Zet de bewerkte pixels terug, bewaar in de cache en geef de ingekleurde mini-canvas terug.
Aanpassen. Donkerder/feller visjes: verlaag L of lift (regel 76). Een heel andere kleur per soort regel je in constants.js (color), niet hier.

1.12 — Teller-animatie & aan/uit met de scroll (regel 190–213)

191-194 // animateCounter: telt in 3800ms soepel op naar 'total' (ease-curve)
201   const observer = new IntersectionObserver(([entry]) => {
202-203   if (entry.isIntersecting) { if (!running && !reduceMotion()) { running = true; tick(); }
204-208     // teller één keer animeren; bij 'animaties uit' meteen eindstand
209   } else { running = false; cancelAnimationFrame(rafId); }
210   }, { threshold: 0.1 });
211   observer.observe(stage);
212   cleanups.push(() => { running = false; cancelAnimationFrame(rafId); …; observer.disconnect(); ro.disconnect(); });
  • 191–194 animateCounter laat het getal in 3,8s soepel oplopen met een ease-curve (eerst versnellen, dan afremmen).
  • 201–203 De IntersectionObserver start de tekenlus zodra het aquarium in beeld is (en animaties aan staan).
  • 204–208 De teller wordt maar één keer geanimeerd (dataset.animated); bij "animaties uit" springt hij meteen naar de eindstand.
  • 209 Scrolt het weg: stop de lus en annuleer het frame (spaart de accu).
  • 210 threshold: 0.1 = "in beeld" zodra 10% zichtbaar is.
  • 212 Opruimwerk: stop alles en koppel observers los. Belangrijk vóór een periodewissel.

rejection samplingNet zo lang een willekeurige plek proberen tot hij aan een eis voldoet (hier: niet te dicht op een andere ping).
jitterEen beetje vaste rommel op een waarde, zodat dingen niet té perfect/regelmatig ogen.
transitionD3 animeert een eigenschap (positie, grootte, kleur) soepel van oud naar nieuw over een tijdsduur.
ease-curveVersnellen/afremmen tijdens een animatie i.p.v. constante snelheid (voelt natuurlijker).
device pixel ratio (dpr)Hoeveel echte pixels één CSS-pixel beslaat (retina = 2). Canvas wordt hierop afgesteld om scherp te blijven.
cleanups / lifecycleLijst met "opruim"-functies. Bij een periodewissel of paginaverlaten worden ze gedraaid zodat niets blijft hangen.
Tot slot. Alle vier de bestanden volgen hetzelfde stramien: data uit state → vorm berekenen → tekenen → interactie koppelen → animeren met raf → opruimen via cleanups. Herken je dat patroon, dan kun je elke grafiek lezen en aanpassen.
